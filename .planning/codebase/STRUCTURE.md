# Directory Structure

## Overview
The repository contains only configuration, hidden initialization folders, and workflow files for the AI assistant.
Application source code has not yet been introduced.

## Key Locations
- `.agent/`: Contains the Get Shit Done (GSD) framework, including active skills, templates, workflows, and tools (`gsd-tools.cjs`).
- `.claude/`: Contains configuration for Claude Code or other compatible AI workflows to work within the environment.
- `.git/`: Version control scaffolding, initialized.
- `.planning/`: Contains project artifacts, config settings, roadmap, and this generated codebase map context.
- `.planning/codebase/`: Contains these generated mapping documents that define the current technical status.

## Naming Conventions
- To be determined by the developers or AI agents during the application's implementation phase.
- Currently, standard Git and hidden folder definitions are used.

## Summary
The current structure is purely administrative and setup-focused. 
Source directories like `src/`, `lib/`, `app/`, `tests/`, or `docs/` will be introduced in upcoming development phases based on tech stack choices.
