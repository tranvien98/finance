# Tech Stack

## Overview
This codebase currently contains no application-level source code. It is an empty workspace initialized with the Get Shit Done (GSD) agent framework. 
The workspace is essentially greenfield for the target application.

## Languages & Runtimes
- **Node.js**: The GSD tools (like `gsd-tools.cjs`) run on Node.js environment naturally.
- **Markdown**: Used extensively for GSD planning and execution context, such as these very files.
- **Bash/Shell**: Used for GSD invocation scripts and automated Git operations.

## Frameworks & Libraries
- None present for the main application. 
- Application-specific frameworks will be determined during the project initialization phase.

## Infrastructure & Deployment
- None present. 
- Deployment strategies and targets will be established once requirements are defined.

## Dependencies
- All current dependencies are encapsulated within the `.agent/` and `.claude/` directories for AI workflow management.
- There are no application `package.json`, `Cargo.toml`, `requirements.txt`, or `go.mod` files present.

## Summary
The stack for the application to be built here is completely undetermined at this point. 
This document should be updated once the project's actual STACK is established via the `/gsd-new-project` research and roadmap phases.
